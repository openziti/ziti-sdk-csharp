# Migration guide: upgrading to the ziti-sdk-c 1.16 native

This release moves `OpenZiti.NET` from the `OpenZiti.NET.native` 1.10.4 native library to 1.16.0. The C SDK
([ziti-sdk-c](https://github.com/openziti/ziti-sdk-c)) changed several model structs, the event model, and a base
numeric type between those versions. This guide covers what changed and what you need to do.

## TL;DR for most apps

If your app uses the modern surface (`ZitiContext`, `ZitiSocket`, enrollment, the `Ziti_*` socket calls), you do not
need to change anything. That path marshals only opaque handles and primitives, so it is unaffected by the struct
changes below. Update the package reference and rebuild.

The breaking changes are confined to the low-level `OpenZiti.Native` interop types (the `ziti_*` structs and event
types). If you P/Invoke those directly, read on.

## Base type change: `model_number` is now 64-bit

In ziti-sdk-c 1.16, `model_number` (the C model integer type) is `int64_t`. Every managed field that maps to a
`model_number` changed from `int` to `long`. This affects:

| Struct | Field(s) now `long` |
| --- | --- |
| `ziti_port_range` | `low`, `high` |
| `ziti_posture_query` | `timeout` |
| `ziti_service` | `perm_flags` |
| `ziti_client_cfg_v1` | `port` |
| `ziti_server_cfg_v1` | `port` |
| `ziti_host_cfg_v1` | `port` |
| `ziti_listen_options` | `connect_timeout_seconds`, `cost`, `max_connections` |

If you read any of these, change your local variables/casts to `long`.

## Struct field changes

- **`ziti_config`**: gained `controllers` (an `IntPtr` to a `model_string` list) between `controller_url` and `id`.
- **`ziti_api_path`**: gained `base_urls` (`IntPtr`).
- **`ziti_api_versions`**: the single `api_path_map` field is replaced by two maps, `edge` and `oidc` (both `IntPtr`).
- **`ziti_version`**: gained `capabilities` (`IntPtr`) before `api_versions`.
- **`ziti_identity`**: `tags` is renamed to `app_data` (still an `IntPtr` to a map).
- **`ziti_posture_query`**: `query_type` changed from a `string` to the `ziti_posture_query_type` enum.
- **`ziti_service`**: gained `terminator_strategy` (the `ziti_terminator_strategy` enum) before `updated_at`.
- **`ziti_intercept_cfg_v1`**: `dial_options_map` is renamed to `dial_options`; gained `allowed_source_addresses`
  (`IntPtr`).
- **`ziti_host_cfg_v1`**: gained `forward_address_translations` (`IntPtr`) and `proxy` (a by-value
  `ziti_proxy_server`).
- **`ziti_options`**: removed `config` and `router_keepalive`; gained `cert_extension_window` (`uint`) and
  `enroll_mode` (the `ziti_enroll_mode` enum).

## Event model is restructured

ziti-sdk-c 1.16 represents events as `ziti_event_t { ziti_event_type type; union { ... } }`. The managed event structs
now model the individual union members directly, so:

- They no longer carry a leading `ziti_event_type` field. Read the event type from the enclosing `ziti_event_t`
  (via `z4d_event_type_from_pointer`) before interpreting the union.
- The `ziti_event_type` enum values were renamed:
  - `ZitiMfaAuthEvent` is now `ZitiAuthEvent`.
  - `ZitiAPIEvent` is now `ZitiConfigEvent`.
- The struct formerly known as `ziti_mfa_auth_event` is now `ziti_auth_event`
  (`{ action, error, error_code, type, detail, providers }`).
- The struct formerly known as `ziti_api_event` (with `new_ctrl_address` / `new_ca_bundle`) is now
  `ziti_config_event` (`{ identity_name, config }`).
- `ziti_context_event` gained `ctrl_count` and `ctrl_details` after `err`.

## Enum value note

Enums generated by ziti-sdk-c's `DECLARE_ENUM` reserve `0` for an "Unknown" sentinel, so their real values start at
`1` (the same convention as `ziti_session_type`, where `Bind = 1`). The managed enums `ziti_posture_query_type`,
`ziti_terminator_strategy`, and `ziti_proxy_server_type` follow this and begin at `1`.

## Platform note for `ziti_options.refresh_interval`

The C field is a `long`, which is 4 bytes on Windows (LLP64) but 8 bytes on Linux/macOS (LP64). The managed
`ziti_options` layout matches Windows. This field is not used on the modern socket runtime path; it matters only if
you marshal `ziti_options` by value on a non-Windows 64-bit platform.
